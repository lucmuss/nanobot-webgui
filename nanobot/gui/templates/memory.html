Syntax Guide

Markdown Help

  • Use # Heading for titles and ## for sections.
  • Use - item for lists and `inline code` for commands or paths.
  • Use fenced code blocks with ``` for prompts, JSON, or shell snippets.
  • Use **bold** for emphasis where the instruction must stand out.
  • Use 1. step lists when the agent should follow a sequence.
  • Use - [ ] task checklists for heartbeat steps or recurring routines.
  • Use block quotes like > note for warnings, assumptions, or handover notes.
  • Prefer short sections with clear headings such as Goal, Constraints, Workflow, and Do not.
  • Use links like [Docs](https://...) when the agent should reference a stable source.
  • Keep secrets out of these files. Store API keys in config or environment variables, not Markdown memory.
